## Q2. IPアドレスとポート番号

このプロジェクトではフロントエンドを `localhost:5173`、バックエンドを `localhost:8080` で起動しています。

- `localhost`とは何を指しますか？
- `:5173` や `:8080` のようなポート番号はなぜ必要ですか？

**回答：**
localhostは自分自身のコンピュータを指す特別な名前です。
ポート番号は一つのコンピュータ上で複数のサービスを区別するために使用します。

## Q3. DNS

ブラウザのアドレスバーに `https://www.example.com` と入力してからページが表示されるまでに、DNSはどのような役割を果たしますか？

**回答：**
DNSはDomain Name Systemの略です。
アドレスバーに表示された文字を通信で実際に扱う番号に変換します。
インターネットの通信には123.123.123.123のようなIP番号と呼ばれる番号を使用します。
しかしこれは人間に分かりづらいため、アドレスバーには文字が表示されるようになっています。
これをIP番号に変換する役割を果たします。
https://www.example.comと入力するとDNSがこれをIPアドレスに変換します。
そのIPアドレスに通信することでWebページが表示されます。

## Q4. HTTPとは

HTTPとは何の略で、どのような役割を持つものですか？「プロトコル」という言葉の意味も含めて説明してください。

**回答：**
HTTPはHyper Text Transfer Protocolの略です。
インターネットで使用される通信ルールの一つで、Web上でブラウザとサーバー間のやり取りを行います。
Hyper Textとはリンクを通じて他のページに移動できる文字列のことです。
通常Textとは異なるという意味で名付けられています。
Protocolはコンピュータ同士が通信を行う場合の決め事を意味します。
つまりHyper Textをやり取りする通信ルールのことです。


---

## Q5. HTTPとHTTPSの違い

HTTPとHTTPSの違いは何ですか？なぜ現在のWebサイトはほぼすべてHTTPSを使用していますか？

**回答：**
HTTPSはHTTPでの通信を暗号化したものです。
HTTPは通信内容を他人から盗み見られる可能性があるため、HTTPSではそれができないように暗号化してデータのやり取りを行います。
個人情報やパスワードなど重要な情報の保護のため、現在のWebサイトでは広くHTTPSが使用されています。

## Q11. CORSとは

**回答：**

ポート番号が異なると別オリジンと認識されるので、セキュリティのためブラウザはリクエストをブロックしますが、これを設定により許可します。
フロントエンドからバックエンドにアクセス可能とするためCORSが必要です。

## Q13. フロントエンドとバックエンド

「フロントエンド」と「バックエンド」の違いを説明してください。このアプリではそれぞれ何を担当していますか？

**回答：**
フロントエンドはユーザーが操作する画面部分で、ユーザーの入力を受け取り、バックエンドにリクエストを送付します。
バックエンドはリクエストの内容を処理し、必要に応じてデータベースとのやり取りを行い結果をフロントエンドに返します。

今回のタスク管理アプリでフロントエンドはカードの表示や作成、更新、削除の操作を行います。
バックエンドがその処理を行い、データベースと必要な情報をやり取りしてフロントエンドに結果を返します。

## Q14. 3層構造

このアプリはフロントエンド・バックエンド・データベースの3層構造です。なぜこのように役割を分けるのですか？1つのプログラムですべてをやらない理由を説明してください。

**回答：**
１つのプログラムで全てを行う場合、何か変更が生じたときにフロントエンド・バックエンド・データベースのそれぞれに変更が生じて複雑になります。
それぞれ独立させることで役割が明確になり、管理しやすくなります。
また、各層を独立させることで再利用性や拡張性が高まり、保守しやすくなります。

## Q15. データの流れ

ユーザーが「新しいタスクを作成」ボタンを押してから、データベースに保存され、画面に反映されるまでの流れを順を追って説明してください。

**回答：**
　ユーザーが「新しいタスクを作成」ボタンを押す
→入力フォームにタスク内容を入力する
→フロントエンドからバックエンドのAPIに対してリクエスト（POST）が送られる

→フロントエンドがその結果を受け取り、画面に新しいタスクを表示する

## Q17. 同期処理と非同期処理

「同期処理」と「非同期処理」の違いを、日常生活の例えで説明してください。なぜWeb通信では非同期処理が必要ですか？

**回答：**
同期処理とは、1つの処理が終わるまで次の処理を待つ方法です。
非同期処理は処理の完了を待たずに次の処理を進める方法です。

例えば料理店で例えると同期処理は注文したら料理が完成するまで待ちます。他のことはできません。
一方非同期処理は注文後に呼び出しベルを受け取り、他のことをしながら料理の完成を待つイメージです。時間を有効活用できます。

Web通信では、サーバーからの応答に時間がかかる場合でも画面全体の処理を止めず、他の操作を続けられるため、非同期処理が必要です。

このアプリはReactで作られたSPAです。従来のWebサイト（ページ遷移するもの）と何が違いますか？

**回答：**

SPA（シングルページアプリケーション）はページ全体の再読み込みを行わず、必要な部分だけを更新するWebアプリケーションです。
従来のWebサイトではページ遷移のたびにサーバーからHTMLを再取得します。
しかし、SPAでは最初に読み込んだページをもとに、APIでデータを取得しながら画面を動的に更新します。
このため、画面遷移が高速でスムーズになる利点があります。

## Q27. トランザクション

トランザクションとは何ですか？タスクを別カラムへドラッグ移動する操作で、なぜ重要ですか？

**回答：**
トランザクションとは、複数のデータベース処理を1つのまとまりとして扱い、すべて成功した場合だけ確定して途中で失敗した場合は元の状態に戻す仕組みです。
タスクを別カラムへドラッグ移動する場合、カードが存在するカラムや並び順など複数のデータを更新する必要があります。
途中で一部の成功した処理だけ確定してしまうと、その先でエラーが出た場合にカードの位置や順番の整合性に問題が生じる恐れがあります。
このため、トランザクションを使って一連の更新をまとめて処理し、データの整合性を保つことが重要です。

## Q28. SQLとORM

- SQLとはどのようなものですか？
- ORMの役割とは何ですか？

**回答：**
SQLはStructured Query Languageの略で、データベースを操作するための言語です。
データの検索、作成、更新、削除などを行うことができます。

ORMはObject Relational Mappingの略で、プログラム上のオブジェクト（データをまとめたもの）とデータベースのテーブルを対応付ける仕組みです。
ORMを使うことで、SQLを直接すべて書かなくても、プログラム上のオブジェクトを通してデータベースを操作できます。
ORMが必要なSQLを自動で生成してくれます。

# 第4章 開発プロセスとチーム開発（4問）

## Q29. バージョン管理（Git）

Gitなどのバージョン管理システムとは何ですか？なぜ必要ですか？使わないとどんな問題が起きますか？

**回答：**
プログラムなどで何か変更があった際に、その履歴を記録・管理する仕組みです。
誰が、いつ、どんな変更を行なったのか確認でき、不具合発生した時に元の状態に戻すことができます。
使わないとどのような変更が行われたのか不明確になり、元の状態に戻せなくなる恐れがあります。
特に複数人で開発を行う場合は変更内容の競合が起きたり、間違って上書きされる恐れがあります。
変更履歴を明確にしておくことが、品質を保ちながら安全に開発を進めるために必要です。

---

## Q30. ブランチとPull Request

- ブランチとは何ですか？
- Pull Request の目的は何ですか？
- なぜ全員が main に直接変更しないのですか？

**回答：**
ブランチとはmainから分岐して作業するための場所です。
mainに影響を与えることなく更新や変更を行うことが可能となります。

Pull Requestはブランチで行った変更をmainに反映する前に、内容をレビューすることが目的です。
不具合や不要な変更が含まれていないか確認することができます。

全員がmainに直接変更を行うと不具合のある内容が反映されたり、競合が起こったり、間違った上書きをされる恐れがあるため通常は実施しません。


---

## Q31. コミットメッセージの重要性

例：`feat: ログイン機能を追加`

良いコミットメッセージを書くことがなぜ重要ですか？

**回答：**
良いコミットメッセージにより、変更の概要が把握できるためです。
コミットメッセージがわかりにくいと内容を読み込まないと変更点がわからず、レビューに時間がかかる、また正しくレビューしにくくなります。
また後から不具合調査や履歴を確認するときも、良いコミットメッセージがあれば確認しやすくなります。

## Q32. なぜ技術的な仕組みを理解すべきか

AI時代においてコードはAIが書けるようになりました。それでもネットワーク・データベース・Webアプリケーションの仕組みを理解する必要がある理由を、あなたの考えで述べてください。

**回答：**
仕組みを理解しておかないとAIが作成したコードが正しいのか、セキュリティに問題はないのか、目的に合っているかなどを判断できないからです。
不具合が発生したときも仕組みを理解しておけば、原因を早期に特定し、適切な修正が行えます。
また内容についてより良い改善を行うこともできます。
このためAIをより良く活用し、品質の高いシステムを作るため必要と考えます。
